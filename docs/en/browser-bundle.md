# Browser Bundle Guide

This document explains how to use Inkshot Engine and its plugins directly in the browser **without installing Node.js**.

---

## Background

`@inkshot/engine` is published as an NPM package for developers with a Node.js workflow. However, **editors or end users should not be assumed to have Node.js installed**.

For this reason, we provide:

| File | Description |
|------|-------------|
| `dist/engine.bundle.js` | ESM file with engine + pixi.js merged (generated by `npm run build:bundle`) |
| `editor/index.html` | Example page using Import Map so all modules share the same engine instance |
| `editor/main.js` | Example entry point showing how to start the engine and load plugins |

---

## Quick Start

### 1. Build the bundle

```bash
npm run build:bundle
# Output: dist/engine.bundle.js  dist/engine.bundle.js.map
```

### 2. Start a local server

ES Modules must be served via HTTP(S) — they cannot be opened directly with `file://`.

```bash
# Any static server works, for example:
npx serve .
# Then open http://localhost:3000/editor/
```

### 3. Game entry point

`editor/main.js` demonstrates the most basic usage:

```js
import { createEngine } from '@inkshot/engine';

const { core, renderer } = await createEngine({
  container: '#app',
  width: window.innerWidth,
  height: window.innerHeight,
});
```

`"@inkshot/engine"` is resolved by the Import Map in `editor/index.html` to point to `dist/engine.bundle.js`.

---

## How Import Maps Work

```html
<script type="importmap">
{
  "imports": {
    "@inkshot/engine": "../dist/engine.bundle.js"
  }
}
</script>
```

All modules that import `"@inkshot/engine"` (including plugins) will reference **the same** `engine.bundle.js`. This ensures:
- Engine code loads only once
- All plugins share the same Pixi.js application instance
- The event bus is a single instance across the entire page

## Plugin Bundling Requirements

For custom plugins to be compatible with this architecture, they must use `@inkshot/engine` as an **external** dependency and output pure ESM format. This way plugin files only contain their own logic without duplicating engine code.

### Using Rolldown

```js
// rolldown.plugin.config.js (in the plugin project)
export default {
  input: 'src/index.ts',
  platform: 'browser',
  output: {
    file: 'dist/my-plugin.js',
    format: 'esm',
    codeSplitting: false,
  },
  external: ['@inkshot/engine'],
};
```

### Using Rollup

```js
// rollup.config.js
export default {
  input: 'src/index.ts',
  output: { file: 'dist/my-plugin.js', format: 'es' },
  external: ['@inkshot/engine'],
  plugins: [/* typescript plugin */],
};
```

### Using Vite (library mode)

```js
// vite.config.js
export default {
  build: {
    lib: { entry: 'src/index.ts', formats: ['es'], fileName: 'my-plugin' },
    rollupOptions: { external: ['@inkshot/engine'] },
  },
};
```

### package.json Requirements

Every `@inkshot/*` plugin package must declare:

```json
{
  "peerDependencies": {
    "@inkshot/engine": "^0.2.0"
  }
}
```

> **Do not** add `@inkshot/engine` to `dependencies` or `devDependencies` and bundle it into the output.

## Loading Plugins

Place bundled plugin `.js` files in the `editor/plugins/` directory, then in `editor/main.js`:

```js
import { createEngine } from '@inkshot/engine';
import myPlugin from './plugins/my-plugin.js';

const { core } = await createEngine({
  container: '#app',
  width: 1280,
  height: 720,
  plugins: [myPlugin],
});
```

You can also dynamically load via URL string (`PluginSource` supports string form):

```js
const { core } = await createEngine({
  plugins: ['https://cdn.example.com/my-plugin.js'],
});
```

## Browser Support

| Feature | Chrome | Firefox | Safari |
|---------|--------|---------|--------|
| ES Modules | 61+ | 60+ | 10.1+ |
| Import Maps | 89+ | 108+ | 16.4+ |
| `modulepreload` | 66+ | 115+ | 17+ |

All modern mainstream browsers are fully supported.
